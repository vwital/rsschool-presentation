KISS (Keep It Simple, Stupid)

It’s a principle in software engineering that suggests that systems work best when they are simple and easy to understand, rather than complex. The idea is to make code easy to read, maintain and debug.

Solving a coding problem with the KISS principle

1. Clearly define the problem and requirements.

2. Break down the problem into smaller, manageable parts.

3. Choose the simplest solution that meets the requirements.

4. Write clean, readable, and well-documented code.

5. Test and debug thoroughly.

6. Refactor and optimize as necessary.

7. Keep the code simple, modular, and maintainable.

How to Apply the KISS principle in Javascript Development?

Write simple, readable code (descriptive variable names, structure your code small and focused, avoid using abbreviations or shortcuts)

Avoid overcomplicating things (use built-in JavaScript functions and libraries, break a big problem into smaller parts, choose the simplest solution)

Keep code organized (consistent coding style, code organized into meaningful blocks using functions, classes and modules.)

Use built-in functions (.forEach(), .reduce(), .filter(), .map(), ...spread, destructuring(arrays, objects))

Minimize dependencies (libraries that are widely used, well-documented, and maintained)

Test and debug regularly(regular testing and debugging, automated testing )

Code Example

Create Simple Functions


				// Avoid
			function sayHello(name) {
				console.log('Hello', name)
			}
				// Use
			const sayHello = name => console.log("Hello", name)

Don't forget: function declaration and arrow functions have different scoping methodology.

Implicit Return


					// Avoid
				function doubleValue(value) {
					return value * 2
				}
					// Use
				const doubleValue = value => value * 2

Destructuring assignment


				const person = {
					first: 'Wes',
					last: 'Bos',
					country: 'Canada',
					city: 'Hamilton',
					twitter: '@wesbos'
				  }
				  // Avoid
				let first = person.first
				let last = person.last
				// Use
				let { first, last } = person

Destructuring assignment


			const fruits = ['apple', 'banana', 'mango', 'pineapple', 'pear']

				// Avoid
			let fruitOne = fruits[0]
			let fruitTwo = fruits[1]

				// Use
			let [fruitOne, fruitTwo] = fruits

Spread Operator


			const odd = [1, 2, 3]
				// Avoid
			const nums = [4, 5, 6].concat(odd)
			  // Use
			const nums = [4, 5, 6, ...odd]
			
			// with Object 
			
			const obj1 = { a: 1, b: 2, c: 3}
				// Use
			const obj2 = { …obj1, d: 4 }

Using Built-in Functions


				const list = [2, 5, 7, 2, 6, 2, 3, 5] 

				// Avoid
				for (let i = 0; i < list.length; i++) { 
					console.log(list[i]) 
				}
				// Use
				list.forEach(number => console.log(number))

Using Built-in Functions


			let distances = [
				{ from: 'New York', to: 'Dhaka', distance: 12654},
				{ from: 'Sydney', to: 'chittagong', distance: 8858},
				{ from: 'Kolkata', to: 'Sylhet', distance: 670}
			]
		

			  // Avoid
			let totalInKilometers = 0
			for(let i = 0; i < distances.length; i++){
  			  	if(distances[i].distance < 10000){
  			  	totalInKilometers += distances[i].distance * 0.621371
 				}
			}
			console.log(totalInKilometers)

				//Use
			let totalInKilometers = distances.filter(item => item.distance < 10000)
								  .map(item => item.distance * 0.621371)
								  .reduce((prev, distance) => prev + distance, 0)
			console.log(totalInKilometers)

Unnecessary complexity


		// Bad
		function calculateCircleArea(radius){
			let PI = Math.PI
			let result = Math.PI * radius * radius
			return result
			}
			
		// Good
		function calculateCircleArea(radius){
			return Math.pi * radius * radius
		}
		// Better
		const calculateCircleArea = (radius) => Math.PI * radius * radius
